## How to View Logs

### Method 1: View Live Logs in Android Studio (RECOMMENDED)

**Step 1: Connect Device or Start Emulator**
- Connect Android phone via USB, OR
- Start Android Emulator

**Step 2: Open Logcat in Android Studio**
- Android Studio → View → Tool Windows → Logcat (or Alt+6)

**Step 3: Filter by Tag**
- In the Logcat filter box, type: `AppDetection`
- This shows only DALE app detection events

**Step 4: Open a Protected App**
- Watch logcat in real-time
- You'll see:
  ```
  📱 APP_OPENED: [App Name] ([Package])
  🔐 LOCK_SCREEN_TRIGGERED: [App Name]
  🔒 APP_CLOSED: [App Name] ([Package])
  ```

## Logcat Events Explained

### App Opened (Protected App)
```
📱 APP_OPENED: WhatsApp (com.whatsapp) from group 'Social Media' [via Accessibility Service]
```
- Shows when a protected app comes to foreground
- Includes app name, package, and group name
- Shows detection method used

### Lock Screen Triggered
```
🔐 LOCK_SCREEN_TRIGGERED ========================================
   App: Instagram
   Package: com.instagram.android
   Group: Social Apps
   Method: Accessibility Service
   Status: User must enter credentials to unlock
🔐 ========================================
```
- Shows detailed lock screen event
- User must enter PIN/Password/Pattern

### App Closed (Protected App)
```
🔒 APP_CLOSED: WhatsApp (com.whatsapp) from group 'Social Media' [via Accessibility Service]
```
- Shows when user leaves the app
- Includes app name, package, and group name

### Accessibility Service Events
```
✅ ACCESSIBILITY_SERVICE_CREATED - Ready to monitor app events
✅ ACCESSIBILITY_SERVICE_CONNECTED - Now listening to app events
📡 Event received - Package: com.instagram.android, Type: WINDOW_STATE_CHANGED
```
- Shows service lifecycle events
- Useful for debugging service issues

### Home Screen / Recents
```
🏠 HOME_SCREEN_OPENED - User on home screen
📋 RECENTS_OPENED - User viewing recent apps
🔄 APP_SWITCHED_FROM_RECENTS - App: com.whatsapp
```
- Shows when user navigates to home or recents
- Helps understand app switching patterns

### Errors
```
❌ ERROR_LOGGING_APP_CLOSED: com.whatsapp - Permission denied
❌ ERROR_IN_CHECK_AND_LOCK: com.instagram.android - null pointer
```
- Shows any errors with clear indicators
- Helps diagnose issues

## All Log Event Types

### 📱 App Opens
- `📱 APP_OPENED:` - Protected app opened
- `✅ UNPROTECTED_APP_OPENED:` - Unprotected app opened (just logged)

### 🔐 Lock Screen
- `🔐 LOCK_SCREEN_TRIGGERED:` - Lock screen shown
- `🔐 LOCK_SCREEN_ALREADY_SHOWN:` - Prevented duplicate lock screen
- `⏳ GRACE_PERIOD_ACTIVE:` - Unlock grace period active
- `🔓 APP_TEMPORARILY_UNLOCKED:` - App unlocked after auth

### 🔒 App Closes
- `🔒 APP_CLOSED:` - Protected app closed

### 🏠 Navigation
- `🏠 HOME_SCREEN_OPENED:` - User on home screen
- `📋 RECENTS_OPENED:` - Recents app opened
- `🔄 APP_SWITCHED_FROM_RECENTS:` - App opened from recents

### 📡 Service Events
- `📡 Event received:` - Accessibility event received
- `⏭️  Skipping package:` - Event skipped (excluded app)
- `✅ ACCESSIBILITY_SERVICE_CREATED:` - Service created
- `✅ ACCESSIBILITY_SERVICE_CONNECTED:` - Service connected

### ❌ Errors
- `❌ ERROR_` - Any error with full details

## Filtering Logcat

### Show Only App Detection Events
```
Filter: AppDetection
```

### Show Only Errors
```
Filter: AppDetection level:ERROR
```

### Show Only App Opens/Closes
```
Filter: AppDetection AND (APP_OPENED OR APP_CLOSED)
```

### Show Accessibility Service Events Only
```
Filter: ACCESSIBILITY_SERVICE
```

---

## Logcat Advanced Filtering

### In Android Studio Logcat:

**Filter for specific app:**
```
package:com.whatsapp AND tag:AppDetection
```

**Filter for time range:**
```
tag:AppDetection time:[10:00 - 10:05]
```

**Filter for specific event type:**
```
tag:AppDetection text:"LOCK_SCREEN_TRIGGERED"
```

## Common Issues & Solutions

### Issue: Not Seeing Any Logs
**Cause:** Accessibility service not enabled
**Solution:**
1. Go to Settings → Accessibility
2. Enable "DALE"
3. Restart app
4. Try again

### Issue: Only Lock Screen Events, No App Open/Close
**Cause:** Events being logged but filter wrong
**Solution:**
1. Clear filter (show all logs)
2. Search for "APP_OPENED" or "APP_CLOSED"
3. Check logcat verbose level

### Issue: Duplicate Events
**Cause:** Multiple detection, this is normal
**Solution:**
- This is expected - accessibility service is working
- If too frequent, may indicate event loop

### Issue: Grace Period Active But App Locked Again
**Cause:** Grace period timing issue
**Solution:**
- Check: `⏳ GRACE_PERIOD_ACTIVE: [Time remaining]`
- Ensure enough time passes before opening again
- Default: 5 seconds grace period

```bash
adb logcat AppDetection:V *:S
```

```bash
adb logcat | grep -i "appdetection\|AccessibilityService"
```

```bash
adb logcat > dale_logs.txt
```

### Clear all logs
```bash
adb logcat -c
```

```bash
adb shell cat /data/data/com.example.dale/files/activity_log.txt
```
